# GREEN SHARKS WEBSITE PROJECT BRIEF
Version 1.0  
Prepared for Sysop Review  

---

# 1. Executive Summary

Green Sharks has grown into a structured, active international Arma community operating:

- AU and US server clusters
- Arma 3 (Vietnam Mike Force, WW2 Mike Force, Zeus Ops)
- Arma Reforger (Modern & Vietnam PvE)
- Cross-group partnerships
- Active YouTube media presence
- Structured rules and CAS doctrine
- No membership fees

The purpose of this project is to design and launch a centralized Green Sharks website that:

- Serves as the public face of the community
- Simplifies onboarding
- Reduces Discord confusion
- Increases AU + US player growth
- Showcases media and legitimacy
- Integrates with Discord where possible

This is not intended to replace Discord.  
Discord remains the operational hub.  
The website becomes the command center and public gateway.

# 3. Community Positioning

Green Sharks should position itself as:

> Structured tactical PvE operations without elitism or contracts.

Key differentiators:

- No fees
- Organized CAS doctrine
- Zeus-guided operations
- Multiple eras (Vietnam, WW2, Modern)
- AU + US server clusters
- International player base
- Community-driven

Tone:

Professional. Tactical. Welcoming. Competent.

Avoid cringe milsim clichés or over-the-top “hardcore realism” branding.

# 6. Discord Integration Strategy

## Phase 1 – Basic Integration

- Discord invite button
- Online member count
- Server preview widget

## Phase 2 – Smart Automation

Create Discord bot that reads:

- #op-schedule
- #announcements
- #server-status

Displays on website:

- Next operation
- Game
- Server
- Brief
- Time (with timezone)

Discord becomes the CMS.

## Phase 3 – Structured Sync

Standardize event post format:

Game:
Server:
Date:
Time:
Brief:
Mods:

Bot parses and auto-updates website.

# 8. Technical Architecture Recommendation

Recommended stack:

- Static site (Astro / Eleventy / Next static export)
- Hosted on Netlify or similar
- JSON-based server data
- Markdown-based guides
- Discord bot for dynamic data
- Optional BattleMetrics embed

No heavy backend required initially.
